Which layout feels right for the test runner?

Card Grid (current style)

Each test is a card in a responsive grid. Click a card to run that single test. "Run All" button at top. Cards flip between pending → running → pass/fail with color. Grouped by profile (AUTH, HANDLERS, SQL, NoSQL, STREAMS, V8).

Compact Table Rows

Each test is a single row in a table. Status icon + test name + duration + run button. Very dense — all 70 tests visible at once. Profile sections are collapsible headers.

Hybrid — Summary Bar + Collapsible Profiles

Top summary bar with pass/fail/total counts and "Run All" button. Below: profile sections (AUTH, HANDLERS, SQL, etc.) that expand to show test cards. Each profile header shows its own pass count and has a "Run Profile" button. Click any card to run individually.
